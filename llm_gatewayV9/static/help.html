Dashboard Help

This service routes LLM calls across seven free providers, with automatic failover, rate-limit tracking, persistent logging, and a manual override. Below: how to use it, what each provider offers, and how to get free keys.

Using the gateway

Python (built-in client)

from client import LLM, ask

# one-shot
print(ask("Explain transformers in 3 lines"))

# explicit provider (shortcut keys: g, n, o, gr, c, or, gh)
print(ask("hi", provider="gr"))   # groq
print(ask("hi", provider="g"))    # gemini

# full client
llm = LLM()
r = llm.chat("hi", provider="gemini", max_tokens=128, temperature=0.5)
print(r["text"], r["provider"], r["latency_ms"])

# streaming
for chunk in llm.stream("count to 5"):
    print(chunk, end="", flush=True)

HTTP (OpenAI-style)

curl http://localhost:8099/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt": "hello", "provider": "g", "max_tokens": 128}'

  • Context: 1M tokens in / 65K out
  • Free tier (Flash-Lite): ~15 RPM · 250K TPM · 1,000 RPD
  • Pro models: 5 RPM · 25 RPD (very tight)

  1. Go to aistudio.google.com/app/apikey
  2. Sign in with your Google account
  3. Click Create API key → pick or create a project → copy the AIza… key

Get a key:

  1. Go to github.com/marketplace/models

7. Ollama (local)

  • Runs on your machine — zero rate limits, zero cost, 0ms cooldown
  • Default: gemma4:31b

Set up:

  1. Install: ollama.com → download for your OS
  2. Pull a model: ollama pull gemma4:31b (or llama3.2, phi4, etc.)
  3. Verify: ollama list
  4. The gateway auto-detects via OLLAMA_URL (default: http://localhost:11434)

Configuration (.env)

The gateway reads from EAGV3/.env. All keys are optional — providers without keys are skipped.

GEMINI_API_KEY=...
GEMINI_MODEL=gemini-3.1-flash-lite-preview

NVIDIA_API_KEY=...
NVIDIA_MODEL=deepseek-ai/deepseek-v3.2

GROQ_API_KEY=...
GROQ_MODEL=llama-3.3-70b-versatile

CEREBRAS_API_KEY=...
CEREBRAS_MODEL=zai-glm-4.7

OPEN_ROUTER_API_KEY=...
OPENROUTER_MODEL=nvidia/nemotron-3-super-120b-a12b:free

GITHUB_ACCESS_TOKEN=...
GITHUB_MODEL=openai/gpt-4.1-mini

LLM_ORDER=ollama,gemini,nvidia,groq,cerebras,openrouter,github
GATEWAY_PORT=8099

How failover works

  1. Caller sends request (with or without provider).
  2. Router estimates token usage from prompt length.
  3. Walks providers in LLM_ORDER, skipping any that:
    • Are inside their per-provider cooldown window (cooldown = 60/RPM)
    • Have hit RPM, RPD, TPM, or daily token cap
    • Have a max_ctx smaller than the prompt
  4. First eligible provider gets the call. On 5xx / timeout / 429 → moves to next.
  5. If provider=... was explicitly set, no failover — error surfaces.
  6. Every call (success or fail) is logged to gateway.db with provider, model, tokens, latency, status.
